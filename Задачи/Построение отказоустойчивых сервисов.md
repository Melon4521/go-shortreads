## Практика по теме "Построение отказоустойчивых сервисов"\*

\* - задачи сгенерированы AI

**Задача 1: Повторные попытки (Retry с backoff)**  
У тебя есть функция, которая делает HTTP-запрос к внешнему сервису, который иногда падает.
Реализуй обертку:

- делает до 3 попыток
- между попытками увеличивает задержку (например: 100мс → 200мс → 400мс)
- если все попытки неудачны → возвращает ошибку
  👉 Важно: не просто цикл, а аккуратная логика retry.

**Задача 2: Таймауты запросов**  
Расширь предыдущую задачу:
Добавь:

- общий таймаут на операцию (например, 2 секунды)
- если exceeded → прекращать все попытки
  👉 Используй `context.Context`.

**Задача 3: Circuit Breaker (упрощённая версия)**  
Реализуй простой circuit breaker:
Поведение:

- если подряд произошло N ошибок → "размыкаем цепь"
- следующие запросы сразу падают
- через некоторое время пробуем снова
  👉 Нужно продумать:
- состояние (closed / open / half-open)
- счётчики

**Задача 4: Ограничение количества запросов (Rate Limiting)**  
Реализуй middleware для HTTP-сервера:
Требования:

- не более N запросов в секунду
- лишние запросы получают `429 Too Many Requests`
  👉 Можно реализовать через:
- token bucket
- или простой таймер

**Задача 5: Ограничение параллелизма (Worker Pool)**  
У тебя есть обработчик, который выполняет "тяжёлую" задачу.
Реализуй:

- не более 3 задач одновременно
- остальные ждут в очереди
  👉 Нужно:
- каналы
- горутины
- контроль нагрузки

**Задача 6: Graceful Shutdown HTTP-сервера**  
Реализуй сервер, который:

- обрабатывает сигналы `SIGINT / SIGTERM`
- перестает принимать новые запросы
- завершает текущие
  👉 Используй `context` + `http.Server.Shutdown`.

**Задача 7: Дедлайн для обработки запроса**  
Добавь в HTTP handler:

- ограничение времени обработки (например, 1 секунда)
- если превышено → вернуть ошибку
  👉 Нужно корректно завершать работу.

**Задача 8: Кеширование результатов (in-memory cache)**  
У тебя есть медленный endpoint.
Реализуй:

- кеш в памяти (map + TTL)
- повторный запрос возвращает кеш
  👉 Нужно продумать:
- инвалидирование
- время жизни

**Задача 9: Дедупликация запросов (singleflight)**  
Если одновременно приходит несколько одинаковых запросов:

- выполняется только один реальный запрос
- остальные ждут результат
  👉 Можно реализовать самому или через `sync/singleflight`.

**Задача 10: Защита от падения (panic recovery)**  
Добавь middleware:

- перехватывает panic
- не падает сервер
- возвращает `500`
  👉 Обязательно логируй ошибку.
