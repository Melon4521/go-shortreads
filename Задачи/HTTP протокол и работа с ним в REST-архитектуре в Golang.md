## Практика по теме "HTTP протокол и работа с ним в REST-архитектуре в Golang"\*

\* - задачи сгенерированы AI

**Задача 1: Мини REST API для заметок**  
Реализуй HTTP-сервер, который хранит заметки в памяти.
Требования:

- `POST /notes` → создать заметку (тело JSON: `{ "text": "..." }`)
- `GET /notes` → получить все заметки
- `GET /notes/{id}` → получить заметку по id
  Дополнительно:
- id генерируется на сервере
- если заметка не найдена → вернуть корректный HTTP статус
  👉 Здесь важно: правильные методы, статус-коды, JSON.

**Задача 2: Фильтрация и query-параметры**  
Расширь API заметок.
Добавь:

- `GET /notes?search=word`
- возвращаются только заметки, содержащие подстроку
  👉 Нужно:
- распарсить query параметры
- аккуратно обработать отсутствие параметров

**Задача 3: Middleware для логирования**  
Добавь middleware, который:

- логирует каждый запрос (`method`, `path`, `status`, время выполнения)
- работает для всех роутов
  👉 Нужно самому продумать обертку над handler.

**Задача 4: Валидация входящих данных**  
Усложни `POST /notes`:
Требования:

- `text` не может быть пустым
- длина не менее 5 символов
  Если данные некорректны:
- вернуть `400 Bad Request`
- JSON с описанием ошибки
  👉 Важно: не просто проверка, а **правильный HTTP-ответ**

**Задача 5: Правильная обработка ошибок (REST стиль)**  
Сделай единый формат ошибок:

```
{
  "error": "message"
}
```

И обеспечь:

- 404 → если ресурс не найден
- 400 → если ошибка клиента
- 500 → если внутренняя ошибка
  👉 Нужно централизовать обработку ошибок.

**Задача 6: Работа с заголовками HTTP**  
Добавь поддержку:

- заголовка `X-Request-ID`
- если его нет → сгенерировать
- вернуть его в ответе
  Также:
- логировать его в middleware

**Задача 7: Таймауты и отмена запроса**  
Реализуй endpoint:

- `GET /slow`
  Он:
- выполняется ~3 секунды
- если клиент отменил запрос → обработка должна остановиться
  👉 Используй `context.Context`

**Задача 8: Клиент для своего API**  
Напиши HTTP-клиент, который:

- создает 3 заметки
- получает их список
- ищет по слову
  👉 Используй `http.Client`

**Задача 9: Параллельные запросы к API**  
С клиента:

- отправь 10 запросов на создание заметок параллельно
- дождись всех результатов
  👉 Тут проверяется:
- конкурентность
- понимание HTTP как сетевого слоя

**Задача 10: Идемпотентность и методы HTTP**  
Добавь endpoint:

- `PUT /notes/{id}` → обновление заметки
  Требования:
- повторный одинаковый запрос не должен менять результат
- если заметки нет → 404
  👉 Нужно понять:
- разницу POST vs PUT
- идемпотентность
