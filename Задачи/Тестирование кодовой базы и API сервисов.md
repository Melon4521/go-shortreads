## Практика по теме "Тестирование кодовой базы и API сервисов"\*

\* - задачи сгенерированы AI

**Задача 1: Табличные тесты для бизнес-логики**  
У тебя есть функция, которая принимает текст заметки и:

- обрезает пробелы по краям
- запрещает пустые строки
- запрещает строки короче 5 символов
  Напиши:
- функцию `NormalizeNote(text string) (string, error)`
- табличные тесты для неё

Покрой случаи:

- нормальный текст
- текст с пробелами
- пустая строка
- слишком короткий текст

**Задача 2: Тестирование ошибок и edge-case’ов**  
Расширь предыдущую функцию:
Добавь правило:

- текст не должен содержать слово `"error"`
  Напиши тесты, которые:
- проверяют, что ошибка возвращается
- проверяют текст ошибки
  👉 Важно: тестировать не только happy path.

**Задача 3: Тестирование HTTP handler (net/http)**  
Протестируй handler:

```
POST /notes
```

Требования:

- отправь JSON-запрос
- проверь статус-код
- проверь тело ответа
  👉 Используй `httptest.NewRecorder` и `httptest.NewRequest`.

**Задача 4: Тестирование негативных сценариев API**  
Для того же endpoint:
Напиши тесты:

- некорректный JSON
- пустой текст
- короткий текст
  Проверь:
- статус = 400
- структура ошибки корректная

**Задача 5: Тестирование middleware**  
У тебя есть middleware логирования или request-id.
Напиши тест:

- отправь запрос через middleware
- проверь, что: - handler вызвался - заголовок добавился
  👉 Нужно протестировать именно обертку.

**Задача 6: Мокирование зависимостей (интерфейсы)**  
Представь, что у тебя есть хранилище:

```
type Storage interface {
	Save(note string) (int, error)
}
```

Требования:

- handler зависит от `Storage`
- в тесте подставь mock-реализацию
  Проверь:
- что handler корректно работает
- что mock был вызван

**Задача 7: Интеграционный тест API**  
Подними реальный HTTP-сервер (через `httptest.Server`).
Напиши тест:

- создай заметку
- получи список заметок
  Проверь:
- что данные действительно сохранились
  👉 Это уже почти production тест.

**Задача 8: Проверка конкурентности**  
Напиши тест:

- отправь 20 параллельных запросов на создание заметок
- дождись выполнения
  Проверь:
- что все заметки сохранились
- нет гонок (запусти с `-race`)

**Задача 9: Покрытие кода и анализ**  
Запусти:

```
go test -cover
```

Требования:

- добейся покрытия ≥ 80%
- найди участок кода без покрытия
- добавь тест
  👉 Нужно осознанно улучшить покрытие.

**Задача 10: Тестирование таймаутов и контекста**  
Протестируй handler:

- который выполняется долго (`/slow`)
  Сделай:
- запрос с отменой через context
- проверь, что handler корректно завершился
