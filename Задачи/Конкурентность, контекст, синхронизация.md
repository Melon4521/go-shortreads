## Практика по темам "Конкурентность и параллелизм. Работа планировщика в Golang" и "Контекст и примитивы синхронизации. Основные паттерны конкурентного кода"\*

\* - задачи сгенерированы AI

**Задача 1: Контролируемый параллелизм (Worker Pool)**
У тебя есть список из 50 задач (например, числа).
Реализуй систему:

- не более 5 задач выполняются одновременно
- каждая задача "работает" ~100мс
- после выполнения возвращает результат
  Выведи:
- результаты всех задач
- общее время выполнения
  👉 Нужно реализовать worker pool через каналы.

**Задача 2: Ограничение скорости (rate limiter)**
Сделай функцию, которая:

- принимает поток задач (канал)
- обрабатывает не более 10 задач в секунду
  👉 Нужно:
- контролировать скорость
- не блокировать всё приложение

**Задача 3: Fan-out / Fan-in паттерн**
У тебя есть функция обработки данных.
Реализуй:

- fan-out: одна очередь задач → несколько воркеров
- fan-in: объединение результатов обратно в один канал
  👉 Важно:
- не потерять данные
- корректно закрывать каналы

**Задача 4: Отмена через context**  
Расширь любую предыдущую задачу:
Добавь:

- `context.Context`
- возможность отмены выполнения
  При отмене:
- все goroutine должны корректно завершиться
  👉 Проверяется понимание graceful cancellation.

**Задача 5: Таймаут обработки**  
Реализуй функцию:

- выполняет задачу
- если она длится больше N времени → отменяется
  👉 Используй:
- `context.WithTimeout`

**Задача 6: Безопасный счётчик (race condition)**  
Есть общий счётчик, который увеличивается из 100 goroutines.
Сделай:

- сначала без синхронизации (поймай race)
- затем исправь: - через `sync.Mutex` - через `atomic`
  👉 Сравни подходы.

**Задача 7: Однократная инициализация (sync.Once)**  
Реализуй ленивую инициализацию ресурса:
Требования:

- ресурс создается только один раз
- даже если вызван из нескольких goroutines
  👉 Используй `sync.Once`.

**Задача 8: Очередь задач с приоритетом**  
Реализуй очередь:

- задачи имеют приоритет (high / low)
- high выполняются раньше
  При этом:
- несколько воркеров обрабатывают очередь
  👉 Нужно продумать структуру + синхронизацию.

**Задача 9: Параллельный pipeline**  
Сделай pipeline из этапов:

1. генерация данных
2. обработка
3. фильтрация
4. вывод
   Каждый этап:

- работает в отдельной goroutine
- передает данные через каналы
  👉 Важно:
- корректное закрытие каналов
- отсутствие утечек goroutines

**Задача 10: Детект и устранение утечки goroutines**  
Создай программу, где:

- goroutines не завершаются (утечка)
  Затем:
- найди проблему
- исправь её
  👉 Подсказка:
- забытый `close(channel)`
- отсутствие cancel
