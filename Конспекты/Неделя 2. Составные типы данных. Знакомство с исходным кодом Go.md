# Массивы

## Синтаксис

**Массив** - тип данных, который содержит фиксированное количество значений определенного типа `T`. В памяти массив представлен последовательной областью ячеек, в которых и хранятся значения массива. Если какие-то значения не указать, то на их месте будет zero-value типа `T`.

```go
var intMass [3]int // <- массив длины 3 из int значений
// [3]int{0, 0, 0}
```

Доступ к элементам массива осуществляется по индексам с нуля. Если обратиться к несуществующему индексу, то получим ошибку.

```go
intMass[0] = 1
intMass[1] = 2

// [3]int{1, 2, 0}
```

#### Другие способы объявления

```go
// Все значения сразу
nums1 := [5]int{1, 2, 4, 5, 5}

// Частичная инициализация значений
nums2 := [5]int{1, 2, 3} // [1 2 3 0 0] остальные нули

// Разреженный массив
nums3 := [4]int{1, 2:5} // [1 0 5 0] мы указали, что под индексом 2 - число 5

// Автоопределение длины
nums4 := [...]{1, 2, 3} // [3]int
```

## Длина и вместительность массива

Есть 2 функции, которые позволяют получить длину и емкость:
- `len` - текущая длина, количество элементов сейчас
- `cap` - вместительность, максимальное количество элементов

Так как в Go у массивов длина фиксированная, то для массива `x`: `len(x) == cap(x)`

## Итерация по массиву

По массиву можно пройтись, используя обычный вариант `for` или `for-range`:

```go
fruits := [3]string{"яблоко", "банан", "апельсин"}

for i := 0; i < len(fruits); i++ {
    fmt.Printf("%d. %s \n", i+1, fruits[i])
}

for index, fruit := range fruits {
    fmt.Printf("%d. %s \n", index+1, fruit)
}

/*
1. яблоко
2. банан
3. апельсин
*/
```

> [!important]
> При передаче массива в функцию передается его копия. Если нужно изменить изначальный массив в функции, то нужна передача по указателю



# Слайсы

## Синтаксис

**Слайсы** - "массивы" без фиксированной длины.

#### Создание слайса

**Вариант 1**. Пустой nil-слайс

```go
var nilSlice []int // nil-слайс под int
```

Пустой слайс не является инициализированным и содержит в себе значение `nil`. Его длина и вместимость по умолчанию 0. Обращение по индексу у такого слайса приведет к ошибке:

```go
fmt.Printf("Длина: %d, Вместимость: %d", len(nilSlice), cap(nilSlice))
// Длина: 0, Вместимость: 0

nilSlice[0] = 1 // panic: runtime error: index out of range [0] with length 0
```

**Вариант 2**. Слайс через литерал `[]T{ ... }`

```go
empty := []int{} // Длина: 0, Вместимость: 0
intSlice := []int{1, 2, 3} // Длина: 3, Вместимость: 3
```

По умолчанию вместимость определяется по его длине

**Вариант 3**. Слайс через `make`

Можно создать слайс через `make([]T, len, cap)`. Если указать ненулевую длину, то значения заполнятся zero-value:

```go
intSlice := make([]int, 2, 3) // []int{0, 0}
```

## Устройство слайса

Под капотом слайс представляется в виде структуры:

```go
type slice struct {
	array unsafe.Pointer // Указатель на первый элемент базового массива
	len int // Количество элементов
	cap int // Текущая вместимость
}
```

Когда мы создаем слайс, то под него создается массив, который хранит в себе элементы, а слайс в себе хранит указатель на этот массив (на его первый элемент).

## Добавление в слайс: `append`

Добавить элементы в слайс можно с помощью функции `append(slice, elems...)`. Эта функция создает **новый слайс** и, грубо говоря, добавляет в него элементы старого + новые элементы и возвращает новый слайс.

```go
intSlice := make([]int, 5, 5) // [0 0 0 0 0]

intSlice = append(intSlice, 1, 2, 3) // [0 0 0 0 0 1 2 3]

fmt.Printf("Длина: %d, Вместимость: %d \n", len(intSlice), cap(intSlice))
// Длина: 8, Вместимость: 10
```

Под капотом `append` работает так:
- Если в текущем слайсе `len < cap`, то в конец базового массива слайса дописываются новые элементы (если их количество удовлетворяет текущей вместимости)
- А если `len == cap`, то под новый слайс создается массив с **удвоенной вместимостью** (как в примере выше) и элементы пишутся в конец.

## Распаковка слайса

Можно распаковать элементы слайса через `slice...`:

```go
func sumVals(vals ...int) int {
	sum := 0
	
	for _, val := range vals {
		sum += val
	}
	
	return sum
}

intSlice = []int{1, 2, 3}

sumVals(intSlice...) // аналогично: sumVals(1, 2, 3)
// 6
```

## Передача слайса в функцию

Мы знаем, что массивы полностью копируются при передаче в функцию. На самом деле в Go все значения копируются, но со слайсами есть нюанс - копируется не сам слайс, а указатель на базовый массив, который он в себе содержит. 

В итоге при передаче слайса в функцию:
- Внутри функции создается совершенно новый слайс с такими же параметрами `len` и `cap`
- В качестве его указателя на базовый массив устанавливается скопированный указатель от переданного слайса
- И когда в функции мы меняем элементы слайса, то по указателю изменяется базовый массив изначального слайса. Поэтому вне функции эти изменения также будут видны:
 
```go
slice := make([]int, 1, 2)

addToSlice(slice)
fmt.Println(slice) // [1] - изменения видны вне функции

func addToSlice(s []int) {
	s[0] = 1
}
```


> [!warning]
> Слайс является несравниваемым типом данных (нельзя сравнить ни с чем другим через `==` и `!=`)

## Реслайсы/срезы

Можно брать новые слайсы на основе других слайсов или массивов. Синтаксис как и в других языках: `slice[start:end]`. Причем указатель нового слайса будет ссылаться на исходный массив или на базовый массив исходного слайса, поэтому изменения будут отражаться и в них тоже. По сути, делая слайс, мы сами выбираем базовый массив для слайса

```go
slice := []int{1, 2, 3, 4, 5}

slice1 := slice[1:3] // [2 3]
slice2 := slice[2:] // [3 4 5]

// меняем в одном
slice2[0] = 500

fmt.Println(slice) // [1 2 500 4 5]
fmt.Println(slice1) // [2 500]
fmt.Println(slice2) // [500 4 5]
```

У нового слайса `capacity` высчитывается как количество элементов от `start` до конца слайса/массива

## Копирование элементов в слайс

Функция `copy(destination, source)` безопасно копирует в `destination` определенное количество элементов из `source`, это количество выбирается как минимальное из `len(destination)` и `len(source)`. Сама функция `copy` возвращает количество скопированных элементов.

```go
// создаем независимую копию слайса
original := []string{"a", "b", "c", "d", "e"}

destination := make([]string, len(original))

fmt.Println("Copied", copy(destination, original)) // 5
fmt.Println(destination) // ["a" "b" "c" "d" "e"]
```


> [!important]
> В Go нет встроенной функции для удаления элемента слайса, поэтому надо реализовывать самостоятельно:
> ```go
> append(slice[:index], slice[index+1:]...)
> ```


# Мапы

## Синтаксис

**Мапа** - тип данных, созданный на основе хэш-таблицы (ключ: значение)

> [!important]
> В качестве ключа можно использовать сравниваемые (`comparable`) типы данных.
> Список несравниваемых (`incomparable`) типов:
> - `slices`
> - `maps`
> - `functions`
> - `struct` с **несравниваемыми** полями

```go
v m map[int]string // nil-мапа с int ключами и string значениями
```

В этом примере объявлена **неинициализированная** мапа с `nil` в качестве значением по умолчанию. При попытке записать что-то по ключу в такую мапу мы получим ошибку.

#### Создание мапы

**Вариант 1**. Объявление через `make(map[T]T, cap)`

Если не указать второй параметр, то создасться пустая мапа, а если указать `cap`, то выделится память под указанное количество значений.

```go
m1 := make(map[int]string) // Пустая инициализированная мапа
m2 := make(map[string]int, 10) // Инициализированная мапа с выделенной памятью на 10 значений
```

**Вариант 2**. Объявление через литерал `map[T]T{key: value, ...}`

```go
m3 := map[float32]bool{1.2: true, 1.5: false} // Инициализированная мапа с фактическими значениями
```

## Работа со значениями, удаление

В мапах доступ к значениям по ключу осуществляется через `[key]`. Удаление через `delete(m, key)`:

```go
m := make(map[int]bool)

m[1] = true // Запись
fmt.Println(m[1]) // true, чтение
delete(m, 1) // удаление
```

## Безопасное получение значения по ключу

Если потенциально пары с указаным ключом может не быть, то можно использовать проверку **value-ok:** `value, ok := m[key]`. Если значение по ключу есть, то мы получим значение и true, а если нет - получим zero-value типа и false:

```go
m := map[int]string{1: "Hello"}

val1, ok1 := m[1] // получили "Hello", true

if !ok1 {
	fmt.Println("Ключа нет") // Вывода не будет
}

val2, ok2 := m[2] // получили "", false

if !ok2 {
	fmt.Println("Ключа нет") // Будет вывод
}
```

## Итерация по мапе

```go
for k, v := range m {  // k - ключ, v - значение. Порядок получения рандомен
    fmt.Println(k, v)
}
```


# Указатели

## Синтаксис

**Указатели** - тип данных, который содержит в себе **адрес** ячейки в памяти какого-то значения или другого указателя. Zero-value указателя - `nil` (ничего, пусто)

При объявлении переменной с указателем в качестве типа мы указываем `*type`, где `type` - тип значения в ячейке памяти

```go
var intPointer *int // nil-указатель на int
fmt.Printf("Тип: %T, значение: %v", intPointer, intPointer)
// Тип: *int, значение: <nil>
```

#### Получение указателя и разыменование

- Чтобы получить указатель от существующего в памяти значения, нужно поставить `&` перед переменной, содержащей это значение
- А чтобы получить исходное значение по указателю нужно поставить `*` перед переменной, содержащей указатель

```go
intVal := 7

var intPointer *int = &intVal

fmt.Printf("%T, %v, %v", intPointer, intPointer,  *intPointer)
// *int, 0xee1a32a60b0, 7
```

>[!warning]
>Если попытаться разыменовать nil-указатель, то получим ошибку:
>
>```go
>var intPointer *int
>fmt.Println(*intPointer)
>// panic: runtime error: invalid memory address or nil pointer dereference
>```

Можно присвоить значение в ячейку памяти (в переменную) по указателю, для этого используем **разыменование** `*`:

```go
intVal := 8
intPointer := &intVal

*intPointer = 7 // изменили значение по указателю

fmt.Println(*intPointer) // 7
```

 Если нужно создать не-nil указатель, но не указывать ему значение (присвоится zero-val), то понадобится `new(type)`:
 
```go
intPointer := new(int)
fmt.Println(*intPointer) // 0

*intPointer = 7
fmt.Println(*intPointer) // 7
```

#### Передача аргументов функций по значению/по указателю

По умолчанию при передаче в функцию какой-то переменной, ее значение просто копируется и при изменении параметра внутри функции, изначальная переменная не поменяется:

```go
num := 5

func square(num int) {
	num *= num // изменяется копия внутри тела функции
}

square(num)
fmt.Println(num) // 5
```

Если мы хотим внутри функции работать с изначальным значеним, то можно передать указатель в функцию и менять переменную по нему:

```go
num := 5

func square(num *int) {
	*num *= *num // разыменовываем
}

square(&num)
fmt.Println(num) // 25
```
