# Точка входа в программу, сборка

## Точка входа

Точкой входа любого проекта в Go является файл с указанием пакета main (`main.go` - принятое название точки входа, но оно необязательное)

Базовая структура `main.go`:

```go
package main

// импорты пакетов
import (
	"fmt"
	// ...
)

func main() {
	// ...
}
```

- `package main` - устанавливаем название пакета main - точки входа в программу
- `func main() { ... }` - основная функция, которая вызывается автоматически при запуске 

## Сборка и запуск

- `go build main.go` - сборка исполняемого файла программы под ОС
- `go run main.go` - быстрая сборка + запуск (юзаем на этапе разработки)
- `go fmt ./...` - форматирование файла

Go можно собирать под разные ОС. Для этого нужно указать переменные окружения перед вызовом `go build`:
- `GOOS` - целевая ОС (windows)
- `GOARCH` - архитектура ОС (amd64)

```go
GOOS=windows GOARCH=amd64 go build main.go
```



# Переменные, типы данных

## Примитивные типы данных

- `bool` - true/false
- `string` - строка, только двойные кавычки
- `int`, `int8`, `int16`, `int32`, `int64` - целые числа, чем больше разрядность типа (число в битах), тем большее число можно хранить и тем больше памяти выделяется
- `uint`, `uint8`, `uint16`, `uint32`, `uint64` - незнаковые целые числа (положительные)
- `byte (uint8)` - байт
- `rune (int32)` - символ в кодировке utf8
- `float32`, `float64` - числа с плав. точкой; `float32` (6–7 значащих цифр) и `float64` (15–17 значащих цифр)
- `complex64`, `complex128` - комплексные числа

> [!important]
> У всех типов есть значения по умолчанию (zero-value):
> `string` "", `int` 0, `bool` false и т.д.

## Объявление переменных/констант

### Переменные

Вариант 1. Указываем **ключевое слово**, **название** и **тип данных**:

```go
var name string
var surname string = "Ivanov"
```

Если сразу инициализируем со значением, то указание типа можно пропустить:

```go
var surname = "Ivanov" // string
```

Присвоение значения:

```go
name = "Ivan"
```

> [!important]
> Если переменной при объявлении не задать значение, то ей присвоится zero-value, которое зависит от её типа

Вариант 2. Множественное объявление

```go
var i, j, k int = 1, 2, 3
var (
	name string = "Ivan" // запятая не нужна
	surname string = "Ivanov"
)
```

Вариант 3. Указываем **название := значение**, тип данных определится по значению

```go
surname := "Ivanov"
```

### Константы

**Константы** - неизменяемые переменные, их объявление совпадает с вариантам 1 и 2 для переменных, только вместо var пишем **const**

```go
const Pi = 3.14159
const name string = "My name"
const surname = "My surname"
const (
	StatusOK       = 200
	StatusNotFound = 404
)
```

### Области видимости

**Область видимости** - область, где переменная доступна.
У функций, условий, циклов есть блоки кода (тело) `{ ... }`, внутри него своя локальная область. Переменные, объявленные внутри блока кода, вне этого блока недоступны.

```go
func hello() {
	var name = "Ivan"
	
	fmt.Println(name) // Ivan
}

fmt.Println(name) // ошибка, name отсюда недоступна
```

Основной поток кода - глобальная область, переменные отсюда видны во всех вложенных блоках кода (в функциях, условиях) при условии, что внутри них не объявлены переменные с таким же именем (затенение):

```go
var name = "Peter"

func hello() {
	var name = "Ivan"
	
	fmt.Println(name) // Ivan
}
```


> [!warning]
> Вне блоков кода (в основном потоке, т.е в глобальной области) способ объявления переменной через **:=** не cработает:
> 
> ```go
> package main
>  
> name := "Ivan" // ошибка
> 
> func main() {}
> ```

## Приведение типов

Для приведения одного типа к другому используются функции, названия которых совпадают с названием нужного типа:

```go
var i int = 15
var f float32 = float32(i)
```

## Сравниваемые и несравниваемые типы

К **сравниваемым** (`comparable`) типам отсносятся:
- `bool`;
- `string`;
- `int`, `int8`, `int16`, `int32`, `int64`;
- `uint`, `uint8`, `uint16`, `uint32`, `uint64`;
- `byte`
- `rune`
- `float32`, `float64`;
- `complex64`, `complex128`;
- `arrays`.

К **НЕсравниваемым** (`incomparable`) типам отсносятся:
- `slices`
- `maps`
- `functions`
- `struct` с **несравниваемыми** полями

# Управляющие конструкции. if, switch

## if/else

Синтаксис условий:

```go
age := 10

if age < 18 {
	fmt.Println("Вы слишком молодой")
} else if age > 80 {
	fmt.Println("Вы слишком старый")
} else {
	fmt.Println("Проходите")
}
```

Короткий синтаксис (объвление переменной + условие):

```go
// if объявление; условие { ... }

if age := 10; age < 18 {
	fmt.Println("Вам сюда нельзя")
} else {
	fmt.Println("Проходите")
}
```

>[!warning]
>При таком способе переменная будет видна только внутри блока `if/else/else if`

Условные операторы:
- `&&` - И
- `||` - ИЛИ
- `!bool` - отрицание

## switch case

Вариант 1. Сравнение значений

```go
switch time.Now().Weekday() {
	case time.Saturday: // значения можно перечислить через ,
		fmt.Println("Today is Saturday")
	case time.Sunday:
		fmt.Println("Today is Sunday")
	default:
		fmt.Println("Today is a working day")
}
```

- Здесь `switch` содержит целевое значение
- Оно поочередно сравнивается со значениями в `case`, если где-то совпало, то выполняется блок кода и работа завершается
- `default` выполняется в случае если ни один `case` не сработал

Вариант 2. Проверка условий

```go
hour := time.Now().Hour()

switch {
	case hour < 12:
		fmt.Println("Good morning!")
	case hour < 17:
		fmt.Println("Good afternoon!")
	default:
		fmt.Println("Good evening!")
}
```

По сути `switch` принимает значение `true`, дальше просто проверяются на истинность условия в `case`.

>[!important]
>В конце блока `case` можно использовать оператор `fallthrough` если нужно выполнить следующий кейс ниже.



# Циклы

В Go все необходимые циклы задаются четырьмя разновидностями `for`:
- Базовый for
- for как цикл while
- бесконечный цикл for
- for-range

## Базовый for

```go
sum := 0

// for объявление; условие; действие { ... }
for i := 1; i < 5; i++ {
	sum += i
}

fmt.Println(sum) // 10
```

Здесь объявленная переменная видна только внутри тела и условия `for`

## For как цикл while

```go
n := 1

// for условие { ... }
for n < 5 {
	n *= 2
}

fmt.Println(n) // 8
```

## Бесконечный цикл for

```go
sum := 0

for {
	sum++
	fmt.Println(sum) // Будем «бесконечно» писать числа в консоль
}
```

## Цикл for-range

Для итерации по перебираемым типам есть следующий цикл:

```go

/*
for индекс, значение := range структура данных {
	// Тело цикла
}
*/

for i, ch := range "hello" {
	fmt.Println("Индекс: ", i, ", значение: ", ch)
}
```

Если требуется итерация только по индексам, то второе значение пропускается. А если нужны только значения, то используем: `_, val := ...`

## break/continue, метки

- `break` - прерывает текущий цикл
- `continue` - прерывает текущую итерацию цикла

Метка цикла позволяет прервать его полностью (break) или пропустить итерацию (continue) из вложенного:

```go
sum := 0

outer:
for {
	sum++
	for j := 0; j < sum; j++ {
		if j*sum == 111 {
			break outer // прерываем верхний цикл
		}
	}
}
```
