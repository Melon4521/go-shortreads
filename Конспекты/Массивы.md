## Синтаксис

**Массив** - тип данных, который содержит фиксированное количество значений определенного типа `T`. В памяти массив представлен последовательной областью ячеек, в которых и хранятся значения массива. Если какие-то значения не указать, то на их месте будет zero-value типа `T`.

```go
var intMass [3]int // <- массив длины 3 из int значений
// [3]int{0, 0, 0}
```

Доступ к элементам массива осуществляется по индексам с нуля. Если обратиться к несуществующему индексу, то получим ошибку.

```go
intMass[0] = 1
intMass[1] = 2

// [3]int{1, 2, 0}
```

#### Другие способы объявления

```go
// Все значения сразу
nums1 := [5]int{1, 2, 4, 5, 5}

// Частичная инициализация значений
nums2 := [5]int{1, 2, 3} // [1 2 3 0 0] остальные нули

// Разреженный массив
nums3 := [4]int{1, 2:5} // [1 0 5 0] мы указали, что под индексом 2 - число 5

// Автоопределение длины
nums4 := [...]{1, 2, 3} // [3]int
```

## Длина и вместительность массива

Есть 2 функции, которые позволяют получить длину и емкость:
- `len` - текущая длина, количество элементов сейчас
- `cap` - вместительность, максимальное количество элементов

Так как в Go у массивов длина фиксированная, то для массива `x`: `len(x) == cap(x)`

## Итерация по массиву

По массиву можно пройтись, используя обычный вариант `for` или `for-range`:

```go
fruits := [3]string{"яблоко", "банан", "апельсин"}

for i := 0; i < len(fruits); i++ {
    fmt.Printf("%d. %s \n", i+1, fruits[i])
}

for index, fruit := range fruits {
    fmt.Printf("%d. %s \n", index+1, fruit)
}

/*
1. яблоко
2. банан
3. апельсин
*/
```

> [!important]
> При передаче массива в функцию передается его копия. Если нужно изменить изначальный массив в функции, то нужна передача по указателю
