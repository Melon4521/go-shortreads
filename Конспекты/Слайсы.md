## Синтаксис

**Слайсы** - "массивы" без фиксированной длины.

#### Создание слайса

**Вариант 1**. Пустой nil-слайс

```go
var nilSlice []int // nil-слайс под int
```

Пустой слайс не является инициализированным и содержит в себе значение `nil`. Его длина и вместимость по умолчанию 0. Обращение по индексу у такого слайса приведет к ошибке:

```go
fmt.Printf("Длина: %d, Вместимость: %d", len(nilSlice), cap(nilSlice))
// Длина: 0, Вместимость: 0

nilSlice[0] = 1 // panic: runtime error: index out of range [0] with length 0
```

**Вариант 2**. Слайс через литерал `[]T{ ... }`

```go
empty := []int{} // Длина: 0, Вместимость: 0
intSlice := []int{1, 2, 3} // Длина: 3, Вместимость: 3
```

По умолчанию вместимость определяется по его длине

**Вариант 3**. Слайс через `make`

Можно создать слайс через `make([]T, len, cap)`. Если указать ненулевую длину, то значения заполнятся zero-value:

```go
intSlice := make([]int, 2, 3) // []int{0, 0}
```

## Устройство слайса

Под капотом слайс представляется в виде структуры:

```go
type slice struct {
	array unsafe.Pointer // Указатель на первый элемент базового массива
	len int // Количество элементов
	cap int // Текущая вместимость
}
```

Когда мы создаем слайс, то под него создается массив, который хранит в себе элементы, а слайс в себе хранит указатель на этот массив (на его первый элемент).

## Добавление в слайс: `append`

Добавить элементы в слайс можно с помощью функции `append(slice, elems...)`. Эта функция создает **новый слайс** и, грубо говоря, добавляет в него элементы старого + новые элементы и возвращает новый слайс.

```go
intSlice := make([]int, 5, 5) // [0 0 0 0 0]

intSlice = append(intSlice, 1, 2, 3) // [0 0 0 0 0 1 2 3]

fmt.Printf("Длина: %d, Вместимость: %d \n", len(intSlice), cap(intSlice))
// Длина: 8, Вместимость: 10
```

Под капотом `append` работает так:
- Если в текущем слайсе `len < cap`, то в конец базового массива слайса дописываются новые элементы (если их количество удовлетворяет текущей вместимости)
- А если `len == cap`, то под новый слайс создается массив с **удвоенной вместимостью** (как в примере выше) и элементы пишутся в конец.

## Распаковка слайса

Можно распаковать элементы слайса через `slice...`:

```go
func sumVals(vals ...int) int {
	sum := 0
	
	for _, val := range vals {
		sum += val
	}
	
	return sum
}

intSlice = []int{1, 2, 3}

sumVals(intSlice...) // аналогично: sumVals(1, 2, 3)
// 6
```

## Передача слайса в функцию

Мы знаем, что массивы полностью копируются при передаче в функцию. На самом деле в Go все значения копируются, но со слайсами есть нюанс - копируется не сам слайс, а указатель на базовый массив, который он в себе содержит. 

В итоге при передаче слайса в функцию:
- Внутри функции создается совершенно новый слайс с такими же параметрами `len` и `cap`
- В качестве его указателя на базовый массив устанавливается скопированный указатель от переданного слайса
- И когда в функции мы меняем элементы слайса, то по указателю изменяется базовый массив изначального слайса. Поэтому вне функции эти изменения также будут видны:
 
```go
slice := make([]int, 1, 2)

addToSlice(slice)
fmt.Println(slice) // [1] - изменения видны вне функции

func addToSlice(s []int) {
	s[0] = 1
}
```


> [!warning]
> Слайс является несравниваемым типом данных (нельзя сравнить ни с чем другим через `==` и `!=`)

## Реслайсы/срезы

Можно брать новые слайсы на основе других слайсов или массивов. Синтаксис как и в других языках: `slice[start:end]`. Причем указатель нового слайса будет ссылаться на исходный массив или на базовый массив исходного слайса, поэтому изменения будут отражаться и в них тоже. По сути, делая слайс, мы сами выбираем базовый массив для слайса

```go
slice := []int{1, 2, 3, 4, 5}

slice1 := slice[1:3] // [2 3]
slice2 := slice[2:] // [3 4 5]

// меняем в одном
slice2[0] = 500

fmt.Println(slice) // [1 2 500 4 5]
fmt.Println(slice1) // [2 500]
fmt.Println(slice2) // [500 4 5]
```

У нового слайса `capacity` высчитывается как количество элементов от `start` до конца слайса/массива

## Копирование элементов в слайс

Функция `copy(destination, source)` безопасно копирует в `destination` определенное количество элементов из `source`, это количество выбирается как минимальное из `len(destination)` и `len(source)`. Сама функция `copy` возвращает количество скопированных элементов.

```go
// создаем независимую копию слайса
original := []string{"a", "b", "c", "d", "e"}

destination := make([]string, len(original))

fmt.Println("Copied", copy(destination, original)) // 5
fmt.Println(destination) // ["a" "b" "c" "d" "e"]
```


> [!important]
> В Go нет встроенной функции для удаления элемента слайса, поэтому надо реализовывать самостоятельно:
> ```go
> append(slice[:index], slice[index+1:]...)
> ```
